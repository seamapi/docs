title: 'Test Accounts'
description: 'Log in to shared sandbox test accounts to connect virtual devices from any manufacturer, so you can build and test without physical hardware.'
---

Seam provides shared **test accounts** for every manufacturer it supports, so you can connect virtual devices in a [sandbox workspace](/core-concepts/workspaces#sandbox-workspaces) and build your integration without any physical hardware.

When you connect a device—in the [Seam Console](https://console.seam.co) or through a [Connect Webview](/core-concepts/connect-webviews)—enter the test-account credentials below instead of real manufacturer credentials.

## The standard sandbox login

Most manufacturers accept the same test-account login:

| Field                              | Value                                                          |
| ---------------------------------- | -------------------------------------------------------------- |
| **Email**                          | Any address in the `example.com` domain—for example, `jane@example.com` |
| **Password**                       | `1234`                                                         |
| **Two-factor / verification code** | `123456` (enter this whenever a device prompts for a code)     |

<Info>
  These are shared, non-secret sandbox fixtures. They work only in [sandbox
  workspaces](/core-concepts/workspaces#sandbox-workspaces) and grant no access
  to real accounts or devices, so they are safe to commit to code and share with
  your team.
</Info>

## Per-brand details

Each manufacturer's [sandbox guide](/device-and-system-integration-guides) lists its exact accepted accounts and the virtual devices pre-seeded with each one. To trigger test events on a virtual device or reset a sandbox workspace, see [Sandbox Workspaces](/core-concepts/workspaces#sandbox-workspaces).
