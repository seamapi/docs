## Overview

Hotek manufactures access control systems designed for the hospitality industry. Their SMART Server platform manages electronic door locks, encoders, and room access across hotel properties, communicating with locks over TCP via a local network.

Seam integrates with Hotek through a [Seam Bridge](/capability-guides/seam-bridge) connection to the Hotek SMART Server. This enables you to manage entrances (rooms and common areas), provision credentials (key cards), and encode plastic cards using Seam's unified API — all without direct interaction with the Hotek system.

<Info>
  Hotek requires a [Seam Bridge](/capability-guides/seam-bridge) installed on the property network to communicate with the Hotek SMART Server. The bridge relays commands between Seam's cloud API and the on-premise Hotek server over TCP.
</Info>

---

## Supported Devices

This integration supports all electronic door locks and encoders connected to a Hotek SMART Server. The Hotek system manages two types of entrances:

- **Guest rooms** — individual hotel room doors (e.g., Room 101, Room 202)
- **Common areas** — shared spaces such as lobbies, gyms, and pools

For detailed information about the Hotek devices that Seam supports, see our [Hotek Supported Devices page](https://www.seam.co/manufacturers/hotek).

## Supported Features

We support the following features:

- Managing ACS users (guests and staff)
- Listing and managing entrances (guest rooms and common areas)
- Creating and revoking credentials (key cards)
- Encoding plastic cards via connected encoders

<Warning>
  Hotek locks do not support remote web unlock. Access is granted by encoding physical key cards or issuing credentials that work with the lock hardware.
</Warning>

## Setup Instructions

To connect a Hotek system to Seam, the property must complete the following steps:

1. Install a [Seam Bridge](/capability-guides/seam-bridge) on the same network as the Hotek SMART Server. The bridge enables communication between Seam's cloud API and the on-premise server.
2. Ensure the Hotek SMART Server is accessible over TCP from the Seam Bridge.
3. Complete the [Seam Connect Webview](/core-concepts/connect-webviews) flow, which will prompt for:
   - **Site Name** — a label to identify this Hotek installation
   - **TCP Port** — the port the Hotek SMART Server listens on
   - **TCP Host** — the IP address of the SMART Server (prefilled to `127.0.0.1` by default)

See the [sandbox credentials page](./sandbox-hotek-access-control-system) for test values to use during development.
